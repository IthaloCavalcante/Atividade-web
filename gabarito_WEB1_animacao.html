transition

🧠 Conceito

  • transition anima a mudança entre dois estados (ex: hover).
  • Sintaxe: transition: propriedade duração timing-function delay;
  • Pode animar múltiplas props separadas por vírgula.
  • Só funciona entre estados definidos (ex: :hover, :focus).
📁 style1.css + index1.html — Botão com transition de cor
CSS
.botao {
  background: royalblue;
  color: white;
  padding: 10px 20px;
  border: none;
  cursor: pointer;
  transition: background 0.3s ease;
  /* propriedade duração timing */
}

.botao:hover {
  background: darkblue;
}

transform

🧠 Conceito

  • transform move/rotaciona/escala/distorce o elemento visualmente.
  • Não afeta o fluxo do documento (não empurra outros elementos).
  • Combina com transition para criar animações suaves.

animation + @keyframes

🧠 Conceito

  • @keyframes nome { ... } define os estados da animação (0%, 50%, 100% ou from/to).
  • animation aplica o keyframe ao elemento.
  • Diferente de transition: roda automaticamente, sem precisar de estado como hover.
  • Sintaxe abreviada: animation: nome duração timing repetições direção;
PropriedadeValores comunsDescrição
animation-namenome do @keyframesQual animação usar
animation-duration1s, 500msQuanto tempo dura
animation-timing-functionease, linear, ease-in-outCurva de velocidade
animation-delay0s, 1sAtraso antes de começar
animation-iteration-count1, infinite, 3Quantas vezes repete
animation-directionnormal, reverse, alternate, alternate-reverseDireção da animação
animation-fill-modenone, forwards, backwards, bothEstado antes/depois da animação
animation-play-staterunning, pausedPausar ou rodar
📁 style6.css — Mover quadrado (position: relative + left)
CSS
.quadrado {
  width: 100px; height: 100px;
  background: red;
  position: relative;

  /* abreviado: nome duração timing repetição */
  animation: mover 2s ease-in-out infinite;
}

@keyframes mover {
  0%   { left: 0; }
  100% { left: calc(100vw - 100px); }
}

📁 style9.css — Balançar (propriedades separadas)
CSS
.caixa {
  width: 100px; height: 100px;
  background-color: coral;
  margin: 100px auto;

  /* Propriedades separadas (ao invés da abreviada): */
  animation-name: balancar;
  animation-duration: 1s;
  animation-timing-function: ease-in-out;
  animation-delay: 0s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-fill-mode: both;
  animation-play-state: running;

  /* equivale a: animation: balancar 1s ease-in-out 0s infinite alternate both running; */
}

@keyframes balancar {
  0%   { transform: rotate(0deg);   }
  25%  { transform: rotate(10deg);  }
  50%  { transform: rotate(-10deg); }
  75%  { transform: rotate(5deg);   }
  100% { transform: rotate(0deg);   }
}

💡 alternate vs normal

  • normal: reinicia do zero a cada ciclo (efeito abrupto)
  • alternate: vai de 0%→100% e volta 100%→0% suavemente (efeito ping-pong)
  • reverse: começa no 100% e vai para 0%

::before e ::after

🧠 Conceito

  • ::before e ::after inserem conteúdo virtual antes/depois do elemento.
  • Exigem content: ""; (mesmo vazio) para existir.
  • São tratados como elementos filhos → aceitam position: absolute.
  • O pai precisa de position: relative para o pseudo-elemento se posicionar corretamente.
  • Podem ser combinados com transition e animation.
📁 style10.css — ::before + transition (linha aparece no hover)
CSS
.animated-button {
  position: relative;     /* âncora o ::before */
  padding: 10px 20px;
  background-color: #4CAF50;
  color: white;
  border: none; cursor: pointer;
  overflow: hidden;       /* esconde o ::before que sai do botão */
}

.animated-button::before {
  content: "";            /* obrigatório! */
  position: absolute;
  bottom: 0; left: 0;
  width: 0;               /* começa com largura zero */
  height: 3px;
  background-color: #fff;
  transition: width 0.3s ease;
}

.animated-button:hover::before {
  width: 100%;            /* expande ao fazer hover */
}

.animated-button:hover {
  background-color: #45a049;
}
📁 style11.css — ::after + @keyframes (linha deslizando)
CSS
.moving-line-button {
  position: relative;
  padding: 10px 20px;
  background-color: #4CAF50;
  color: white;
  border: none; cursor: pointer;
}

.moving-line-button::after {
  content: "";
  position: absolute;
  bottom: 0;
  left: -100%;    /* começa fora do botão, à esquerda */
  width: 100%;
  height: 3px;

  0%   { left: -100%; }  /* fora à esquerda */
  50%  { left: 50%;   }  /* meio do botão   */
  100% { left: 100%;  }  /* sai à direita   */
}

clip-path

🧠 Conceito

  • clip-path recorta o elemento em uma forma geométrica — o que estiver fora da forma fica invisível.
  • Pode ser animado com transition e @keyframes.
  • Formatos: circle(), inset(), polygon(), ellipse().
FormatoSintaxeDescrição
circle()circle(50% at 50% 50%)Círculo: raio at posição-X posição-Y
inset()inset(top right bottom left)Recorte retangular interno
polygon()polygon(x1 y1, x2 y2, ...)Forma livre com vértices
ellipse()ellipse(rx ry at cx cy)Elipse

  );

:target

🧠 Conceito

  • :target ativa quando o ID do elemento corresponde ao fragmento (#id) da URL.
  • Ex: clicar em <a href="#card"> → URL vai para #card → .card:target { ... } ativa.
  • Para "fechar", usar <a href="#"> que remove o fragmento.
  • Permite mostrar/esconder elementos sem JavaScript.

📝 Sintaxe abreviada — animation

animation: nome  duração  timing    delay  repetições  direção   fill-mode  play-state;
animation: girar   1s      linear    0s      infinite   alternate    both      running;

/* Mínimo necessário: */
animation: nome duração;
animation: girar 1s;

📝 @keyframes — estrutura

@keyframes nomeDaAnimacao {
  0%   { /* estado inicial */ }
  50%  { /* meio */           }
  100% { /* estado final */   }
}

/* Ou com from/to (= 0% e 100%): */
@keyframes girar {
  from { transform: rotate(0deg);   }
  to   { transform: rotate(360deg); }
}

📝 clip-path — referência rápida

clip-path: circle(raio at X Y);
  circle(0% at 50% 50%)    /* invisível */
  circle(150% at 50% 50%)  /* tudo visível */

clip-path: inset(top right bottom left);
  inset(0 100% 0 0)  /* 100% recortado da direita = escondido */
  inset(0 0 0 0)     /* sem recorte = visível */

clip-path: polygon(x1 y1, x2 y2, x3 y3, ...);
  polygon(50% 0%, 100% 50%, 50% 100%, 0% 50%)  /* losango */

📝 ::before / ::after — estrutura obrigatória

.pai {
  position: relative;  /* obrigatório se o pseudo-elemento usa position: absolute */
  overflow: hidden;    /* opcional: esconde o pseudo-elemento que sai dos limites */
}

.pai::before {
  content: "";         /* OBRIGATÓRIO — sem isso o elemento não existe */
  position: absolute;
  /* ... demais estilos ... */
}

/* Combinar com hover: */
.pai:hover::before { width: 100%; }

/* Combinar com @keyframes: */
.pai::after { animation: moveLine 2s infinite; }
