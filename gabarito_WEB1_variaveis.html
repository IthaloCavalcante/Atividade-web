O que são variáveis CSS?

🧠 Conceito

  • Variáveis CSS se chamam Custom Properties (propriedades customizadas).
  • São declaradas com dois hífens: --nome-da-variavel: valor;
  • São usadas com a função var(--nome).
  • Normalmente declaradas em :root para ficarem disponíveis em todo o documento.
  • Diferente de variáveis SASS/LESS: as CSS são dinâmicas — mudam em tempo real no navegador.
  • Herdam valor do elemento pai (são afetadas pelo escopo do DOM).

Sintaxe completa

📝 Estrutura básica

/* 1. Declarar no :root (disponível globalmente) */
:root {
  --cor-primaria: #2ecc71;
  --espaco:       20px;
  --borda-radius: 10px;
  --fonte:        'Arial', sans-serif;
}

/* 2. Usar com var() */
.elemento {
  background-color: var(--cor-primaria);
  padding:          var(--espaco);
  border-radius:    var(--borda-radius);
  font-family:      var(--fonte);
}

/* 3. Fallback: segundo argumento de var() */
color: var(--cor-inexistente, black);
/*                            ↑ usado se a variável não existir */

/* 4. Reutilizar a variável em cálculos */
margin: calc(var(--espaco) * 2);
padding: calc(var(--espaco) / 2);

💡 Por que usar variáveis aqui?

  • O valor 20px é usado em 3 lugares: padding do body, gap e padding das caixas.
  • Com variável, basta mudar --espaco em um lugar e tudo atualiza.
  • Sem variável, precisaria alterar manualmente os 3 lugares — risco de inconsistência.

Dark Mode Toggle — só com CSS

📁 style.css + exemplo-cores.html — Troca de tema com variáveis + checkbox hack

Olá, mundo!

Tema trocado só com CSS!

CSS — style.css (completo)
/* Variáveis padrão (tema claro) */
:root {
  --cor-fundo: white;
  --cor-texto: black;
}

/* Esconde o checkbox visualmente mas mantém funcional */
#tema {
  position: absolute;
  opacity: 0;
  width: 0;
  height: 0;
}

/* Quando marcado → redefine as variáveis no irmão .conteudo */
#tema:checked ~ .conteudo {
  --cor-fundo: black;
  --cor-texto: white;
}

/* Usa as variáveis (mudam automaticamente com o :checked) */
.conteudo {
  background-color: var(--cor-fundo);
  color: var(--cor-texto);
  min-height: 100vh;
  display: flex;
  flex-direction: column;
  align-items: center;
  justify-content: center;
  transition: background 0.5s, color 0.5s;
}

/* ── Estilo do switch (slider) ── */
.slider {
  width: 50px;
  height: 24px;
  background-color: #ccc;
  border-radius: 24px;
  position: relative;
  transition: background 0.3s;
}

/* bolinha dentro do slider */
.slider::before {
  content: "";
  position: absolute;
  width: 20px; height: 20px;
  left: 2px; top: 2px;
  background-color: white;
  border-radius: 50%;
  transition: transform 0.3s;
}

/* quando marcado: muda a cor do slider... */
#tema:checked + label .slider {
  background-color: #666;
}

/* ... e move a bolinha */
#tema:checked + label .slider::before {
  transform: translateX(26px);
}
HTML
<!-- checkbox que fica escondido -->
<input type="checkbox" id="tema">

<!-- label clicável = o switch visual -->
<label for="tema" class="switch">
  <div class="slider"></div>
</label>

<!-- conteúdo que muda de tema -->
<div class="conteudo">
  <h1>Olá, mundo!</h1>
  <p>Tema trocado só com CSS!</p>
</div>

💡 Como funciona o truque?

  • O <input type="checkbox"> fica invisível (opacity: 0) mas ainda funciona.
  • O <label for="tema"> está vinculado ao checkbox → clicar no label marca/desmarca.
  • O seletor #tema:checked ~ .conteudo usa o seletor irmão ~ para atingir o .conteudo quando o checkbox está marcado.
  • Dentro do .conteudo, as variáveis --cor-fundo e --cor-texto são redefinidas.
  • Como o background-color e color usam var(), eles atualizam automaticamente.
  • O transition no .conteudo garante a animação suave entre os temas.
SeletorO que seleciona
#tema:checked ~ .conteudoO .conteudo que é irmão do #tema quando marcado
#tema:checked + label .sliderO .slider dentro do label imediatamente após o #tema marcado
~ (til)Seleciona todos os irmãos seguintes

Escopo das variáveis

🧠 Conceito

  • Variáveis CSS respeitam o escopo do DOM — a variável é herdada pelos filhos do elemento onde foi declarada.
  • Um elemento filho pode sobrescrever a variável do pai sem afetar outros elementos.
  • :root = escopo global (disponível para todos).
  • Uma classe ou ID específico = escopo local (só afeta aquele elemento e seus filhos).
🔬 Demo — escopo e herança de variáveis
.pai → --cor: vermelho
.filho → --cor: azul (sobrescreveu)
.neto → herda --cor: azul do pai
CSS
/* Global */
:root {
  --cor: gray; /* padrão global */
}

/* Redefine no .pai */
.pai {
  --cor: red;
  color: var(--cor); /* vermelho */
}

/* Redefine no .filho (não afeta .pai) */
.filho {
  --cor: blue;
  color: var(--cor); /* azul */
}

/* .neto herda --cor: blue do .filho */
.neto {
  color: var(--cor); /* azul (herdado) */
}

Fallback — valor padrão

🧠 Conceito

  • var(--nome, fallback) — se a variável não existir ou for inválida, usa o fallback.
  • O fallback pode ser qualquer valor CSS, inclusive outra variável.

📝 Exemplos de fallback

/* Variável não definida → usa "red" */
color: var(--cor-inexistente, red);

/* Variável não definida → usa outra variável */
color: var(--cor-primaria, var(--cor-secundaria));

/* Cadeia de fallbacks */
color: var(--a, var(--b, var(--c, black)));

/* Fallback com calc() */
padding: var(--espaco, 16px);
margin:  calc(var(--espaco, 16px) * 2);

📝 Diferença: variáveis CSS vs SASS/LESS

/* SASS — estático, compilado, não muda no runtime */
$cor: red;
.box { color: $cor; }

/* CSS Custom Properties — dinâmico, muda em tempo real */
:root { --cor: red; }
.box  { color: var(--cor); }

/* Com CSS é possível fazer: */
document.documentElement.style.setProperty('--cor', 'blue');
/* e TODOS os elementos que usam --cor atualizam instantaneamente */
